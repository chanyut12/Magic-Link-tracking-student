# แผนวิเคราะห์ช่องว่างระบบ (Gap Analysis Plan) - อ้างอิงตาม SRS และ RTM

จากการตรวจสอบ Source Code (API ของ NestJS) สอดคล้องกับเอกสาร `Software Requirements Specification (SRS)` และ `Requirements Traceability Matrix (RTM)` โดยตรง (ไม่นำ Usecase มาเป็นหลัก) พบแผนงานและสถานะดังนี้:

---

### 📝 คำอธิบายสัญลักษณ์ความคืบหน้า:
- ✅ `[x]` = **สร้างโครงสร้างและ API ครบถ้วนแล้ว** (มี Endpoint รองรับชัดเจน)
- 🟡 `[/]` = **มีบางส่วนแล้วแต่ยังไม่สมบูรณ์** (In Progress ต้องปรับปรุงหรือเพิ่มฟิลด์ตาม SRS)
- ⚪ `[ ]` = **ยังไม่ได้เริ่มสร้างฟีเจอร์นี้เลย**

---

## 🔐 1. Identity and Access Management (ระบบจัดการผู้ใช้และสิทธิ)
- [x] **FR-AUTH-01:** เข้า/ออกระบบ และยืนยันตัวตน (มี API `/users/login` แล้ว)
- [x] **FR-AUTH-02:** จัดการบัญชีผู้ใช้ เพิ่ม/แก้ไข/ลบ/ค้นหา (มี CRUD พื้นฐานในระบบ Users แล้ว)
- [x] **FR-AUTH-06:** สร้างลิงก์มอบหมายงานแบบ Dynamic และส่งต่อ (มี API `/task/:token/delegate` แล้ว)
- [/] **FR-AUTH-07:** บันทึกและยืนยันการลงพื้นที่ (มี API การ Tracking ด้วยลิงก์ OTP และ Verify แล้ว **แต่อาจต้องเสริมการเช็คด้วย ThaiID ตาม RTM ถ้าระบบพร้อม**)
- [ ] **FR-AUTH-03:** กำหนดสิทธิ์ตามกลุ่ม (Role) และระดับข้อมูล (มี Role `users/roles` แต่ยังไม่ได้ทำระบบจัดการกั้นสิทธิ์ข้อมูลแบบ Dynamic เต็มรูปแบบ)
- [ ] **FR-AUTH-04:** จัดการกลุ่มเมนูและสิทธิ์การเข้าถึงเมนู
- [ ] **FR-AUTH-05:** แก้ไขข้อมูลส่วนตัว (Profile) และเปลี่ยนรหัสผ่าน

## 👩‍🎓 2. Student Info (ข้อมูลนักเรียนและการเช็คชื่อ)
- [x] **FR-STD-05:** เช็คชื่อเข้าเรียน (มีระบบ Attendance Controller ระดับหนึ่งแล้ว)
- [/] **FR-STD-01:** แสดงรายชื่อนักเรียนแยกตามปีการศึกษาและเทอม (สามารถดึงด้วย API พื้นฐานได้ แต่ยังไม่มีระบบจัดการภาพรวมทั้งหมด)
- [/] **FR-STD-02:** แสดงรายละเอียดข้อมูลรายบุคคล (ดึงได้จาก History แต่อาจขาดหน้า Profile ของผู้เรียนเอง)
- [ ] **FR-STD-03:** แก้ไขข้อมูลที่อยู่ปัจจุบัน หรือพิกัดที่อยู่ของนักเรียน
- [ ] **FR-STD-04:** นำเข้า/ส่งออกข้อมูลประชากรเด็กและนักเรียน (Excel/CSV) ผ่านระบบจัดการ (ตอนนี้ต้องรัน Script มืออยู่)
- [ ] **FR-STD-06:** บันทึกแบบประเมิน DQ/SDQ (ตนเอง/ครู/ผู้ปกครอง)
- [ ] **FR-STD-07:** อนุญาตให้นักเรียนดูข้อมูลโปรไฟล์และสถานะของตนเอง

## 📍 3. Tracking (ระบบติดตามตัวและรายงานผล)
- [x] **FR-TRK-02:** มอบหมายงานให้ผู้เกี่ยวข้องลงพื้นที่ติดตาม (มีใน Task Control)
- [/] **FR-TRK-03 & FR-TRK-04 & FR-TRK-05:** (รวบยอดการติดตาม, ลงหลักฐานแนบรูป, และบันทึกช่วยเหลือส่งต่อ) - **ระบบปัจจุบันมี Flow การส่งข้อมูลกลับ (`/task/:token/submit`) แต่ยังต้องตรวจสอบว่าข้อมูลภาพถ่าย/เหตุผลตรงตามฟอร์มใหม่ของ SRS หรือไม่**
- [x] **FR-TRK-06:** ตรวจสอบข้อมูลและอนุมัติ "ปิดเคส" (มี `/cases/:caseId/review` แล้ว)
- [x] **FR-TRK-01:** แจ้งเตือนผู้อำนวยการอัตโนมัติเมื่อขาดเรียน 3 วัน (ทำระบบ Automation/Cronjob รองรับการเช็คจัดสร้างเคสแล้ว)

## 🗄 4. Master Data (ข้อมูลพื้นฐาน)
- [ ] **FR-MST-01:** จัดการข้อมูลพื้นฐาน (สถานศึกษา, เขตพื้นที่)
- [ ] **FR-MST-02:** จัดการข้อมูลปัจจัยเสี่ยง และสาเหตุการหลุดจากระบบ
- [ ] **FR-MST-03:** จัดการข้อมูลมาตรการช่วยเหลือและหน่วยงานที่เกี่ยวข้อง

## 🔗 5. Integration (การเชื่อมต่อและดึงข้อมูลภายนอก)
- [ ] **FR-INT-01 ถึง FR-INT-04:** เชื่อมโยงข้อมูลผ่าน API กลาง (เช่น กะทรวงสาธารณสุข และ OBEC)

## 📊 6. Audit Log & Report
- [/] **FR-RPT-01:** แสดงแดชบอร์ดรายงานภาพรวม (มี `/stats/overview` แต่หน้าตา UI น่าจะยังไม่ได้ประกอบร่าง)
- [ ] **FR-LOG-01 ถึง FR-LOG-03:** บันทึกประวัติ Login, Access, และ Data Change (Log ระบบ ยังขาดโครงสร้างที่จะนำมาแสดงและ Export ผลชัดเจน)
- [/] **FR-NOT-01:** ศูนย์รวมแจ้งเตือน (Notifications Center) สำหรับคอยเตือนเด็กเสี่ยงหรือข่าวสาร (มีแถบกระดิ่งแจ้งเตือนเคสพร้อมนับยอดแบบเรียลไทม์แล้ว)

## User Review Required

> [!IMPORTANT]
> **ระบบปัจจุบันเน้นโครงกระดูกหลักของ Flow ธุรการการติดตามผู้เรียนไว้ให้หมดแล้ว (Identity ยืนยันผ่าน OTP, ส่งลิงก์ทำ Task แบบกระจายศูนย์, ดึงผลส่งกลับ, ผอ. อนุมัติเคส)** 
> 
> หากเราโฟกัสเฉพาะ **SRS/RTM** และให้ความสำคัญกับสิ่งที่ยังทำมาไม่ครบ เราเริ่มทำทีละส่วนได้ดังนี้ครับ:
>
> 1. **Phase 1 (ปิดจบ Flow ติดตามให้บริบูรณ์):** ไล่เปรียบเทียบตารางเก็บข้อมูล "การส่งผลติดตาม (FR-TRK-03 ถึง 05)" และระบบ "การแก้ไขที่อยู่นักเรียน (FR-STD-03)" แล้วเพิ่มระบบอัปโหลดแนบรูปให้เรียบร้อย
> 2. **Phase 2 (วางระบบแวดล้อมเพื่อจัดการง่ายขึ้น):** ทำหน้า Master Data (FR-MST-*) สำหรับแอดมินใช้เพิ่มโรงเรียน/เขต และข้อมูลปัจจัยเสี่ยงต่างๆ + ระบบ Profile ล็อกรหัส
> 3. **Phase 3 (ระบบติดตามอัตโนมัติ):** ทำสคริปต์แจ้งเตือนเด็กขาด 3 วัน (FR-TRK-01) และเชื่อมรายงาน (FR-RPT)
>
> **อยากให้ผมเริ่มดำเนินการตาม Phase 1 เลยไหมครับ หรือมีฟีเจอร์ตัวไหนที่อยากทำก่อนเป็นพิเศษไหมครับ?**
